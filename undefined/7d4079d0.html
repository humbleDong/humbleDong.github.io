SpringBoot日志管理

概述

SpringBoot框架内部提供了几种日志框架的启动器：

日志框架 启动器
Java Util Logging spring-boot-starter-logging(默认)
Logback spring-boot-starter-logging
Log4j2 spring-boot-starter-log4j

日志格式

日志主要包含以下信息：

  • 日期时间：精确到毫秒，易于排序
  • 日志级别：文章后面单独介绍
  • 应用进程ID
  • 分隔符(–)：用来区分实际开始的日志
  • 线程名称：
  • 日志名称：一般值日志出的当前类的缩写
  • 日志具体内容

控制台日志

SpringBoot默认从控制台输出日志，默认情况下，会记录ERROR，WARN，INFO级别的日志，如果要让应用输出更多日志，可以在应用启动时使用–debug标志来启动调试模式

注：

  • 启动应用的Debug调试模式，并不是让应用输出所有DEBUG级别的日志，而是让应用输出更多的框架核心日志，框架核心日志包括嵌入式容器，SpringBoot等组件的更多日志
  • 另外，还可以开启应用的trace追踪模式，日志要比Debug调试模式更详细

日志文件

SpringBoot默认是把日志输出到控制台

将日志输出到日志文件，可以设置这两个属性

配置参数 说明
loggin.file.name 指定需要输出日志文件名称
logging.file.path 指定需要输出日志文件路径

其原理就是在SpringBoot主包的自动配置文件中注册一个LoggingApplicationListener监听器，然后监听ApplicationEnvironmentPreparedEvent事件

然后initialize初始化方法中通过LogFile#get方法获取具体的日志文件

这个ApplicationEnvironmentPreparedEvent是在PreparedEnvironment方法准备环境阶段时发布的

SpringBoot在获取日志配置的时候为什么不是用@ConfigurationProperties注解绑定配置而是手动获取配置呢？这是因为日志是在ApplicationContext创建之前初始化的，所以此时还无法获取环境中的配置

日志归档

SpringBoot的日志达到默认的10MB之后就会自动归档，如果使用的是默认的Logback日志框架则可直接在Application配置文件中配置日志滚动规则，其他日志框架需要在其日志配置文件中配置(如Log4j2)

Logback日志归档的配置参数对应参数类LogbackLoggingSystemProperties

日志滚动配置参数 说明
logging.logback.rollingpolicy.file-name-pattern 创建日志归档的文件名模式
logging.logback.rollingpolicy.clean-history-on-start 应用启动是否进行日志归档清理
logging.logback.rollingpolicy.max-file-size 日志归档文件的最大文件容量，超过就会进行归档(默认10MB)
logging.logback.rollingpolicy.total-size-cap 日志归档可占用的最大容量，超过就会删除旧的归档日志文件
logging.logback.rollingpolicy.max-histoty 日志归档文件要保留的最长天数(默认7天)

日志配置文件*

概述

SpringBoot默认加载的日志配置文件：

日志框架 配置文件
Logback logback-spring.xml logbakc-spring.groovy logback.xml logback.groovy
Log4j2 log4j2-spring.xml log4j2.xml
Java Util Logging logging.properties

SpringBoot建议使用*-spring格式的配置文件，因为标准号的日志配置文件初始化太早，会导致Spring无法完全控制日志的初始化,*-spring格式的好处就是可以使用Spring中的日志拓展功能

但是可以通过SystemProperty系统参数的方式进行配置。
